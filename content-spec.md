# Content Spec

## Observational and Quick Series

Posts in the following series are artifacts of thinking, optimized for momentum and honest snapshots over polish.

File naming convention:
- **Format:** `YYYY-MM-DD-slug.md`
- **Slug rules:** Include series name + specific topic, single dashes only.
- **Date:** Use publish date.
- **Examples:** `2026-01-07-notes-from-children-of-ash-and-elm.md`, `2026-01-08-decision-point-xtdb-vs-postgres.md`, `2026-01-09-project-postcard-gleanmo-charts.md`, `2026-01-10-design-notes-terminal-density.md`

### Notes from...

- **Purpose:** Capture takeaways from books, movies, essays, talks, or periods of reflection.
- **Tone:** Reflective, observational, non-exhaustive.
- **Title format:** `Notes from <source or context>`
- **Tags:** `notes` (required); optional `books`, `movies`, `history`, `personal`, etc.

### Decision Points

- **Purpose:** Document trade-offs and choices made under real constraints.
- **Tone:** Practical, honest, non-performative.
- **Title format:** `Decision Point: <choice A> vs <choice B>` or `Decision Point: <framing of the decision>`
- **Tags:** `decision-points` (required); optional domain tags like `databases`, `gamedev`, `tools`, etc.

### Project Postcards

- **Purpose:** Share visual or concrete snapshots of projects in progress.
- **Tone:** Casual, present-tense, status-oriented.
- **Title format:** `Project Postcard: <project> <what exists now>`
- **Tags:** `project-postcards` (required); optional project tags like `gleanmo`, `gamedev`, etc.

### Design Notes

- **Purpose:** Capture visual taste, UI observations, or aesthetic reactions.
- **Tone:** Subjective, descriptive, image-forward.
- **Title format:** `Design Notes: <what stood out>`
- **Tags:** `design-notes` (required); optional `ui`, `typography`, `layout`, etc.

### Data Visualizations

- **Purpose:** Exploratory views into personal metrics or data experiments.
- **Tone:** Sketchy, provisional, curious.
- **Title format:** `Data Visualization: <what is being explored>`
- **Tags:** `data-visualizations` (required); optional `gleanmo`, `tracking`, `personal-data`, etc.
